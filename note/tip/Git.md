# Git: 버전관리 program

``` 작동원리, Git bash가 무엇인지, Git bash 명령어 ```

1. 작동원리 다른 프로그램들은 마지막꺼만 저장하는데 Git은 하나하나 과거의 수정했던 이력들을

   저장한다.

   같은파일을 동시에 작업할때 그냥 push를 하면 문제가 생기니 status를 저장하고

   commit -m " " 을 보내 구별할수있게 하고 서로 합칠수있게 만들었다.

2. Git bash란 리눅스 bash 명령어를 윈도우에서 쓸수있게 해주는것

   Git명령어를 윈도우에서 쓸수있게 해주는것

   MINGW64에서 리눅스명령어를  윈도우에서 쓸수있게 해주는것

3. git add는 작업 디렉토리(working directory)에서 준비 영역(staging area)로 보내는 것이다.

   git commit -m " " 는 HEAD로 보내는최종확인 명령어다. commit message에는 수정사항을 적고보내는 명령어다.

   git push는 HEAD에서 github으로 보내는것이다.

   git status는 현재상태를 확인하는것이다.
