# os(operating system)

```:  구조,  계정, 권한, 파일 수정 과정, 종류 ```

1. 구조에는 하드웨어(H/W), 소프트웨어(S/W), os가 있다 소프트웨어는 os에 맞는걸 써야한다.

2. 계정에는 관리자계정과 사용자계정이 있다 이렇게 나눠서 사용하는 이유는 보안때문이다

   해커들이 관리자계정을 주로 공격하기때문에 관리자계정에 읽기, 쓰기, 실행권한을 부여해두고

   비밀번호를 아주 어렵게 설정해야하는데 국가표준비밀권장사항으로 비밀번호는 12자리이상 대문자, 소문자, 숫자, 특수문자를섞어쓰도록 국정원에서 정해두었는데 관리자계정이 해킹당하면 컴퓨터자체가 해킹당했다는 소리란것이기때문이다.

3. 파일수정과정에는 보통사람이 생각하는것에는 그 파일하나에서 수정된다고 생각되겠지만

   컴퓨터 내에서는 3가지파일이 있어야 수정이된다

   A 원본파일 B 내용수정한파일 **  C 비어있는파일(temporary ) ** 은 꼭 있어야한다

   A파일을 C파일에넣고 B파일을 A파일에넣은뒤 C에 들어간 A파일을 B파일로 넣은뒤

   C파일을 삭제한다 이 과정에서 용량이 1.5배 늘어나지만 삭제하면되니 크게 상관은 없다.

   단순하게보이지만 제일 중요한것이다.

4. 종류에는

   windows, unix(macOS,iOS), linux(cromOS)가 있다.
